    简要介绍

    本篇笔记是针对剑指offer-第十题的多种不同解法和衍生题型的笔记。

    解法一、递归

    因为斐波那契数列的公式定义为f(n+1) = f(n)+f(n-1),很直接会想到套娃

    class Solution:
    def fib(self, N: int) -> int:
        if N <= 1:
            return N
        return self.fib(N-1) + self.fib(N-2)

    优点：直观 缺点：耗时巨大

    在堆栈中我们需要与 N 成正比的空间大小。该堆栈跟踪 fib(N) 的函数调用，随着堆栈的不断增长如果没有足够的内存则会导致 StackOverflowError

    解法二、记忆化自底向上

    class Solution:
    def fib(self, N: int) -> int:
        if N <= 1:
            return N
        return self.memoize(N)

    def memoize(self, N: int) -> {}:
        cache = {0: 0, 1: 1}

        # Since range is exclusive and we want to include N, we need to put N+1.
        for i in range(2, N+1):
            cache[i] = cache[i-1] + cache[i-2]

        return cache[N]

    方法三、记忆化自顶向下的方法

    class Solution:
    def fib(self, N: int) -> int:
        if N <= 1:
            return N
        self.cache = {0: 0, 1: 1}
        return self.memoize(N)

    def memoize(self, N: int) -> {}:
        if N in self.cache.keys():
            return self.cache[N]
        self.cache[N] = self.memoize(N-1) + self.memoize(N-2)
        return self.memoize(N)

    小总结：
    方法二和方法三都是使用额外的空间->字典来存储已经计算过的元素，等下次再用的时候不用一步一步算回去

    方法四、自底向上进行迭代👍

    思路是用两个指针 prev1和prev2分别记录倒数第二和倒数第三，然后用这俩计算倒数第一

    计算好倒数第一后，再让倒数第三指向倒数第二，倒数第二指向倒数第一，再进行下一次运算

    class Solution:
    def fib(self, N: int) -> int:
        if (N <= 1):
            return N
        if (N == 2):
            return 1

        current = 0
        prev1 = 1
        prev2 = 1

        # Since range is exclusive and we want to include N, we need to put N+1.
        for i in range(3, N+1):
            current = prev1 + prev2
            prev2 = prev1
            prev1 = current
        return current%1000000007

作者：LeetCode
链接：https://leetcode-cn.com/problems/fibonacci-number/solution/fei-bo-na-qi-shu-by-leetcode/

    时：O(N) 空：O(1)

    其实第四种方法是来自于509. 斐波那契数这里面并没有写return current%1000000007，只是用的return current，而509没事，剑指 Offer 10- I. 斐波那契数列必须%1000000007是因为其底下的例子非常多是0 <= n <= 100，而在509底下标明0 ≤ N ≤ 30

    那么为什么大数在这里就不行了呢？为什么非要除以1e9+7呢？

    参考https://www.liuchuo.net/archives/645和https://www.zhihu.com/question/49374703

    得原因如下：

    • 虽然该数并不是最大的小于2^30的质数，但是好记且是质数

    • 对于int32来说，1e9+7已经足够大了，可以刚好使得两数相加不会爆int，相乘不会爆long long，使得计算出来的结果在32位有符号整数内

    • int64位的最大值为2^63-1，对于1000000007来说它的平方不会在int64中溢出
      所以在大数相乘的时候，因为(a∗b)%c=((a%c)∗(b%c))%c，所以相乘时两边都对1000000007取模，再保存在int64里面不会溢出

    方法四与动态规划的思想十分相像，转移方程为f(n) = f(n-1)+f(n-2)

    https://leetcode-cn.com/problems/fei-bo-na-qi-shu-lie-lcof/solution/mian-shi-ti-10-i-fei-bo-na-qi-shu-lie-dong-tai-gui/

    class Solution:
    def fib(self, n: int) -> int:
        a, b = 0, 1
        for _ in range(n):
            a, b = b, a + b
        return a % 1000000007

    873. 最长的斐波那契子序列的长度

    思路是将给定数组中的每两个数都组成一对儿，然后分别看这两个数的和是否在数组中，再改变两个“指针”，并使得计数器+1，再继续判断

    class Solution:
    def lenLongestFibSubseq(self, A: List[int]) -> int:
        s = set(A)
        n = len(A)
        result = 0
        for i in range(n-1):
            for j in range(i+1,n):
                a,b = A[i],A[j]
                count = 2
                while a+b in s:
                    a,b = b,a+b
                    count+=1
                    result = max(result,count)
        return result if result>2 else 0

    842. 将数组拆分成斐波那契序列

    解法一、暴力

    class Solution(object):
    def splitIntoFibonacci(self, S):
        for i in range(min(10, len(S))):
            x = S[:i+1]
            if x != '0' and x.startswith('0'): break
            a = int(x)
            for j in range(i+1, min(i+10, len(S))):
                y = S[i+1: j+1]
                if y != '0' and y.startswith('0'): break
                b = int(y)
                fib = [a, b]
                k = j + 1
                while k < len(S):
                    nxt = fib[-1] + fib[-2]
                    nxtS = str(nxt)
                    if nxt <= 2**31 - 1 and S[k:].startswith(nxtS):
                        k += len(nxtS)
                        fib.append(nxt)
                    else:
                        break
                else:
                    if len(fib) >= 3:
                        return fib
        return []

    306. 累加数

    class Solution:
    def isAdditiveNumber(self, num: str) -> bool:
        if len(num) < 3: return False
        result = []

        def dfs(begin, path, left_num):
            if len(path) > 2 and left_num == 0: # 数字被用完
                result.append(path)
                return
            for i in range(begin + 1, len(num) + 1):
                if i > begin + 1 and num[begin] == '0': return # 多位数字不能以0开头
                if len(path) < 2:
                    dfs(i, path + [int(num[begin:i])], len(num) - i)
                else:
                    if int(num[begin:i]) == path[-1] + path[-2]:
                        dfs(i, path + [int(num[begin: i])], len(num) - i)

        dfs(0, [], len(num))
        return  bool(result)

    剑指 Offer 10- II. 青蛙跳台阶问题

    跳台阶可以看作是斐波那契数列的变种,只不过在这里注意0阶也是1种跳法

    class Solution:
    def numWays(self, n: int) -> int:
        a,b = 1,1
        if n==0:
            return 1
        if n==1:
            return 1
        while n>0:
            a,b = b,a+b
            n-=1
        return a%1000000007

    746. 使用最小花费爬楼梯

    在这里只记录第二种解法（第一种有点绕）

    class Solution:
    def minCostClimbingStairs(self, cost: List[int]) -> int:
        n = len(cost)
        dp = [0] * n
        dp[0], dp[1] = cost[0], cost[1]
        for i in range(2, n):
            dp[i] = min(dp[i - 2], dp[i - 1]) + cost[i]
        return min(dp[-2], dp[-1])
