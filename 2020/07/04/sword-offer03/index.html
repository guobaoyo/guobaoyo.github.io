    剑指offer_第三题&a,b = b,a备注

    简要介绍

    本篇笔记是针对剑指offer-第三题的多种不同时间空间复杂度的解法，主要是整理评论和题解中感觉比较优雅的解法，且这里包括Python中的 a,b = b,a的备注笔记

    解法

    出自评论区中普通人能想到的解法和旧坟新绿以及Krahets大神的题解

    1 排序比较法 时O(nlogn),空O(1)

    class Solution:
    def findRepeatNumber(self, nums: List[int]) -> int:
        nums.sort()
        for i in range(len(nums)-1):
            if nums[i] == nums[i+1]:
                return nums[i]

    2 哈希表 时O(n)，空O(n)

    相对于第一种方法属于用空间换时间

    class Solution:
    def findRepeatNumber(self, nums: List[int]) -> int:
        repeatDict = {}
        for num in nums:
            if num not in repeatDict:
                repeatDict[num] = 1
            else:
                return num

    3 原地交换 时O(n)，空O(1)

    class Solution:
    def findRepeatNumber(self, nums: [int]) -> int:
        i = 0
        while i < len(nums):
            if nums[i] == i:
                i += 1
                continue
            if nums[nums[i]] == nums[i]: return nums[i]
            nums[nums[i]], nums[i] = nums[i], nums[nums[i]]
        return -1

    在一个长度为 n 的数组 nums 里的所有数字都在 0 ~ n-1 的范围内。

    上述条件是第三种方法成立的前提，因为数字都在0~n-1 所以就可以通过

    nums[nums[i]], nums[i] = nums[i], nums[nums[i]]

    使得nums[i]放到其应在的位置，且在交换之前有判断条件：

    if nums[nums[i]] == nums[i]: return nums[i]

    如此判断可以确保如果nums[i]已经正确就不用再交换

    注意在这里的交换操作只能使用上述顺序，而不能使用： 

    nums[i], nums[nums[i]] = nums[nums[i]], nums[i]

    在https://blog.csdn.net/qq_43029747/article/details/95992657 文章中详细说明了a,b = b,a的原理是

    在python中，会在过程中生成一个元组 c，并且c = (b ,a)，然后进行a = c[0] ， b = c[1] 的操作

    在例子array = [1,2,3,4,5,6,7]中

    如果使用array[x],array[array[x]] = array[array[x]],array[x] 生成一个(array[array[x]],array[x])值为 (2,1)

    • 让a的值等于c[0]:array[x] = 2
    • 让b的值等于c[1]:array[array[x]]的值等于1，此时array[x]已经等于2相当于给第三个值赋值为1

    因此其结果为[2, 2, 1, 4, 5, 6, 7]

    而如果使用array[array[x]],array[x] = array[x],array[array[x]] 生成一个c:(array[x],array[array[x]])值为(1,2)

    • 让a的值等于c[0]即array[array[x]]=1 即 array[array[0]] = 1

    • 让b的值等于c[1]即array[x]等于2 而在这里x的值没有改变，所以不会出错

    4 二分法 时O(nlongn),空O(1),且不改原数据

    注：第四种方法对本题并不适用

    但是适用的题目与本题类似，适用条件为“长度为n的数组，且数字都在1~n-1之间，且一定有数字是重复的”

    且第四种方法的优点是在空间复杂度为1的前提下且不修改原数据下，找到重复数字

    缺点是不能保证找出所有的重复的数字

    class Solution:
    def findRepeatNumber(self, nums:List[int]) -> int:
        # 注意初始值是1
        min_value = 1
        max_value = len(nums) - 1
        while (max_value > min_value):
            mid_value = (max_value + min_value) // 2
            counts = self.countNums(nums, min_value, mid_value);
            if counts > mid_value - min_value + 1:
                max_value = mid_value
            else:
                # 注意这个地方需要加1，不然最后会陷入死循环，比如最后max_value为2,min_value为1，则会一直循环,对应的上面也可以给max_value复制的地方减1
                min_value = mid_value + 1
        # 跳出循环的条件一定是max_value = min_value
        return min_value

    def countNums(self, nums, min_value, max_value):
        count = 0
        for ele in nums:
            if min_value <= ele <= max_value:
                count += 1
        return count
