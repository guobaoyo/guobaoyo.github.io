    主要为强化学习的基础介绍

    一、关于强化学习

    交叉与区别点

    从图中可以看出强化学习是一个与其他学科有着很多交叉的知识领域，在计算机学科中，强化学习属于机器学习的这一大范畴内，在工程学中，强化学习与优化控制理论有着很大交叉等等。

    我们主要的切入方向是从计算机这个方向去切入，按照学习模式可以将机器学习分为三种，分别是监督学习，无监督学习，强化学习。在监督学习中，每个数据样本都有一个标准答案，且把这些样本输入到网络时有两个特点

    • 这些数据都是符合独立同分布条件
    • 标答也给到网络去计算loss

    而训练强化学习智能体时，训练数据是一个交互过程（一个时间序列）

    • 智能体所观测到的状态不符合独立同分布条件，上一帧和下一帧是十分相关的
    • 在训练的时候不会告诉智能体去做哪个动作，智能体需要自己去与环境进行交互，在最大化累计奖励的过程中去寻找最优策略
    • 奖励（或者说反馈）具有延迟性，也就是智能体当前时刻做出了个动作并不会马上获得奖励，有可能在多步之后再获得奖励，因此强化学习的训练也会更加困难
    • 基于第二点，在强化学习中也不能单凭奖励值的大小去判断这个动作决策的好坏，因为在强化学习中还有个比较重要的概念就是探索，智能体有可能会舍弃掉即时奖励比较大的动作，而去选择即时奖励较小的动作，其根本原因还是第二点虽然当前这个动作的即时奖励较小，但是可能在后面所带来的延迟奖励会很大

    从相互比较的角度来看，监督学习中图像分类任务中的标答都是人给定的标准答案，所获得效果最好就是让训练出来的模型无限逼近于人类的图像分类能力。 而强化学习的这种学习范式是让智能体在环境中自己探索，所以在某些环境下有可能会达到超越人类的结果。

    例子

    所以近年来尤其是游戏策略方面，在强化学习方面产出了非常多杰出的工作，如很多Atari游戏中比人类玩得更好，alpha go打败围棋顶尖高手李世石，AlphaStar在星际2中达到了Grandmaster级别
    王者荣耀人工智能“绝悟”击败顶尖职业玩家。

    二、强化学习基本要素

    奖励

    首先，在强化学习中的一个非常标志性的元素是奖励值reward，reward是一个标量的反馈信号，该信号表示在时间步t 智能体做出该动作后环境反馈给智能体关于这个动作的好坏程度。在强化学习问题中，解决的目标可以描述为最大化累计奖励。在具体的例子中，比如基于强化学习控制的无人机控制系统，如果该飞机随着期望轨迹飞行，则增加奖励，如果飞机坠机或者没有按照预期飞行，则减少奖励值。在atari游戏中，如果分数增加则增加奖励值，如果分数减少则减少奖励值。而智能体的目的就是最大化累计奖励，其目的是选择一定的行为序列以最大化未来的总体奖励这些行为可能是一个长期的序列，奖励可能而且通常是延迟的，有时候宁愿牺牲即时（短期）的奖励以获取更多的长期奖励。

    环境

    具体区别中的第二点则是强化学习问题中都会有一个动态变化的环境，一般情况下是智能体通过接收环境输出的观测值$O_t$ 执行动作$A_t$，环境负责接收智能体执行的动作$A_t$改变其环境观测值$O_{t+1}$ 并且产生奖励值$R_t$ ,智能体再根据新的环境观测值和奖励值再输出相应动作$A_{t+1}$

    智能体与环境交互所产生的这个序列可以看作一个历史序列,标识为history

    智能体实际上是根据历史序列信息来选择动作，而非单个环境观测值。

    解释状态，智能体状态，环境状态，信息状态的关系

    状态$S_t $ 可以分为以下三种，分别是环境状态，智能体观测到的状态，信息状态

    • 环境状态是环境的私有表示，环境状态对agent不是可见、看不懂的，agent不能直接依靠环境状态来执行action。

    • 智能体状态是agent的内在表示，agent state是可以被智能体理解的状态，agent state是历史的函数。

    • 信息状态也叫马尔可夫状态包含了历史序列的所有的有用信息，如果一个状态是马尔可夫状态，说明具有Markov性质，未来和过去是独立的，未来状态仅仅取决于当下的状态$S_t$，当你得到状态$S_t$后，只需要将$s_t$存储在这里，具有Markov性质，就可以扔掉history的整个的其他部分，因为history对你起到的帮助比不上state，因为状态决定未来该采取什么行动，状态可以代表整个的history，这样更加精简，是一种更好的方式。

    当环境状态=智能体状态=观测值说明这个环境是Fully Observable Environments 且该过程为是个马尔科夫决策过程

    以蛇棋为例，在这个游戏中，环境状态就等于智能体可观测到的状态是一个Fully Observable Environments

    当Agent state 不等于 environment state 时说明这是个部分可观察马尔科夫决策问题 ，以星际争霸或者英雄联盟为例，这些游戏中具有战争迷雾，其中一方作为智能体不可能看到所有的状态信息，所以Agent state 不等于 environment state 是部分可观测的马尔可夫决策问题

    三、智能体组成部分

    一个RL agent可能包含下列一个或多个因素：其中Policy（策略）：agent 的行为函数。输入为状态，输出为行动决策。第二个是value function（价值函数）：评估每个状态或行动有多好。第三个是Model（模型）：agent 对环境的表示，agent眼里的环境，可以用于判断环境的变化。

    首先，策略指的是state到action的映射，指的是智能体的行为函数，其中策略还分为确定性策略和随机性策略，确定性策略指的是输入这个状态后得到确定的动作。

    而随机策略指的是输入这个状态后得到的是随机性策略，这个动作是符合一定的概率分布采样得到的，同一状态输入到模型中得到的动作可能是不同的。

    其次，价值函数指的是对未来奖励的累计预测值，这个预测值可以用来评估状态的好坏，智能体可以根据价值函数来选择不同的行为。

    最后是模型Model，这个model是用来模拟环境的模型，主要由两部分组成，分别为状态转移矩阵和奖励机制，状态转移矩阵的作用是用来预测下一个状态，预测环境的动态变化，奖励机制用来反馈给智能体即时奖励。

    进而可以从智能体的内置函数类型对智能体进行分类：

    • 基于价值的智能体->内含价值函数
    • 基于策略的智能体->内含策略函数
    • actor-critic->内含价值函数和策略函数

    从智能体是否对环境进行建模可以分为

    • model free智能体
    • model based智能体

    四、强化学习相关问题

    探索和利用

    在强化学习中需明确第一对儿概念是探索和利用的关系

    探索指的是对于已掌握的信息中可以放弃掉某些奖励，为的是得到未来更大的奖励

    利用指的是利用已知信息来最大化回报，开发利用已有的信息

    需要注意的是探索和利用同等重要，不能单单偏向其中一个

    举例来讲，去一个你最喜欢的饭店就是利用（Exploration），而去一家新的饭店则是探索exploration，探索这家新的饭店有可能会非常好吃，刷新顾客的认知，代替原来的那家“最好”的饭店。也有一定的风险性，就是这家店做饭难吃，带给顾客不好的体验。

    预测和控制

    预测问题指的是输入给定的状态输出这个状态的价值函数，来“预测”这个状态的价值是多少

    控制问题的目标是找到这个状态下最好的动作
