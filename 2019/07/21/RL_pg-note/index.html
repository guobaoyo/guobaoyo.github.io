    改进二、添加折扣因子

    按照上述公式计算该序列的总回报奖励是有待改进的，改进点是添加折扣因子，其目的是有两个，一是在没有终止状态的游戏环境中，如果不用折扣因子会使得回报值G趋近于无穷大，而添加折扣因子后，使得回报值G有界。添加折扣因子的第二种理解角度是，对于同等数量奖励值，添加(0,1)的$\gamma$可以更看重近期奖励。

    改进三、添加偏置值（基线baseline）

    存在的问题：不论决策如何，最终的长期回报值都为正，即我们增强所有策略（对实际效果不好的策略的提升程度有所降低）是不合理的，应该降低不好的行动的概率，而不是轻微提升不好的行动的概率。所以应给所有时刻的长期累计回报减去一个偏移量，用b表示“baseline”

    基于值函数方法的局限性

    基于值函数的方法主要有以下几个局限性

    • 对连续动作的处理能力较差
      • 在基于值函数的算法中，需要先计算出下一时刻状态下所有动作的价值，并从中选出最优的行动价值，如果动作数量是有限的，这是可行的，而如果动作空间连续，无法进行这种计算和选择。
      • 策略梯度法直接对轨迹的价值期望求导，不需要进行最优行动的选择，因此连续型动作空间的问题可以使用策略梯度算法求解。
    • 通常不好解决随机策略问题
      • 基于值函数的强化学习方法对应的最优策略通常是确定性策略，因为其是从众多行为价值中选择一个最大价值的行为，而有些问题的最优策略却是随机策略。
      • 以走迷宫为例，智能体无法区分迷宫中的灰色格子，只能对灰色格子产生一种策略（都向左走或者都向右走）这种确定性策略会导致智能体卡在图中的某个灰色格子中。

    总结：policy-based的优缺点？

    答：https://www.bilibili.com/video/BV1fZ4y1x7mp?from=search&seid=2347657740900666455

    • 优点为
      • 无论数据的多少，我们都会得到一个策略函数（数据的多少决定是全局最优还是局部最优）
      • 在高维的动作空间更有效（相比于value-based方法不需要输出非常多的动作个数的神经元，只是输出概率分布即可）
      • 学得的是随机策略在某些情况下会比确定性策略更有效（可以举迷宫的例子）
      • 从表达形式可以看出策略梯度的核心思想是求得目标函数对于策略$\theta$的导数，而目标函数可以写为关于状态的价值函数，一个episode的总奖励值等，通过梯度上升的方法迭代求出目标函数的极大值。

    • 缺点：

      • 收敛到局部最优解
      • 方差较大（reward波动非常大）

    policy-based和value-based算法的主要区别是什么？

    • 神经网络输出（或者说学的东西）不一样
      • value-based算法输出的是各个动作的Q值（学的是价值函数，通过价值函数得到策略 ）
      • policy-based输出的是动作的概率分布（不学价值函数，直接学策略）
